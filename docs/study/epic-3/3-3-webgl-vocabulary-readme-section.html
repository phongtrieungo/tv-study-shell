Whiteboard in ≤2 minutes

  1. Buffer — GPU memory for verts. createVbo + per-paint bufferData(DYNAMIC_DRAW) of Visible Window quads (render.ts).
  2. Texture — GPU image sampling. createColorAtlas → texImage2D (texture.ts).
  3. Shader / program — VS maps pixels to clip; FS samples the atlas. createShaderProgram (shaders.ts).
  4. Draw call — one batched drawArrays(TRIANGLES, …) in drawVisibleWindow — not one call per cell; not drawElements in this lab. 
context (webgl2 → webgl)
  → program + VBO + atlas
  → upload verts for Visible Window only
  → drawArrays (batched)
  → leave → disposeGpu   ←── VRAM ≠ JS GC

Batching talking point

Same shared Visible Window math as Canvas EPG. Canvas pays many 2D ops per paint; WebGL rebuilds a small interleaved VBO (x,y,u,v,focus) and issues one draw call for all visible quads. Cite the measured compare in canvas-vs-webgl.md — trade-offs, not “GPU always wins.”
