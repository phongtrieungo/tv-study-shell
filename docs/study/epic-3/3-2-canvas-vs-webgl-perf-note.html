Canvas vs WebGL · Side-by-side Perf Note

Same Visible Window idea, two renderers, one labeled machine. Interview proof lives in docs/perf-notes/canvas-vs-webgl.md — not “WebGL always wins.”

Mental model

  1. Why compare (docs/webgl-investment.md) — Canvas teaches virtualization; WebGL teaches the GPU pipeline. Blits/Lightning is applied WebGL later. Measuring both closes the “I only used a framework” gap.
  2. Same math, different renderer — both Surfaces import computeVisibleWindow from @tvshell/shared. Windowing is a data problem; Canvas 2D vs WebGL is a renderer choice.
  3. AD-8 — Perf Notes under docs/perf-notes/ with browser / OS / machine class labeled. README links them. Unlabeled FPS does not count.
  4. Paint model — both coalesce paints on focus / resize (not a continuous full-grid RAF loop). Idle compositor FPS ≈ display refresh; do not confuse that with “we paint 120 grids per second.” 
shared VW math → drawn ≪ logical 600   ←── both Surfaces

Canvas 2D path → many CPU 2D ops / paint
WebGL path     → buffers + texture + shaders
                 → 1× batched drawArrays / paint

How to measure (same session)

  1. Run Shell → Menu → EPG.
  2. Read HUD: drawn N ≪ logical 600. Arrow around; note the drawn band.
  3. Back → Menu → WebGL Lab.
  4. Read HUD: drawn N ≪ logical 600 · context webgl2|webgl. Arrow around.
  5. Optional: DevTools Command Menu → Show Rendering → Frame rendering stats; optional CPU throttle 4×–6× (testing-strategy.md).
  6. Write / cite docs/perf-notes/canvas-vs-webgl.md with one shared Environment block for both Surfaces.

Whiteboard talking points

  • “Both Surfaces virtualize — on this laptop stage the HUD showed drawn 60 ≪ logical 600 for Canvas and WebGL.”
  • “WebGL’s Visible Window paint ends in one batched drawArrays; Canvas pays many 2D ops. That is draw-call / CPU framing, not a blanket FPS win.”
  • “I will not claim TV FPS from a MacBook without naming the proxy. Domain floor is ~30 FPS UI; 60 is aspirational on entry hardware.”
  • “Next story (3.3) names the pipeline pieces — buffers, textures, shaders, draw calls — using this lab as the example.”
