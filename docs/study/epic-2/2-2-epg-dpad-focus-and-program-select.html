TV Study Shell · Story 2.2 · FR-5

EPG · App-owned D-pad focus + program select

Canvas grids have no DOM tab order. Focus is an app cursor (channelIndex, timeMs), painted as a visible ring, moved with getDpadAction, and used to open a minimal program detail on Enter.

Mental model

  1. App-owned focus — not document.activeElement and not CSS :focus on cells. One cursor drives both the amber highlight and the Visible Window.
  2. Axes — ↑↓ change channel (keep the same focus time = column affinity). ←→ jump to the adjacent program on that channel (program-boundary nav), not a fixed 30-minute scrub.
  3. Select — Enter → select → resolve program with half-open [start, end) → show a small DOM panel (title + time).
  4. Back stack — detail open → dismiss detail; detail closed → Shell leaves the Surface. Never steal the second Back. 
keydown (capture)
  getDpadAction(event)
        │
        ├─ detail open + back  → close detail (stopPropagation)
        ├─ detail open + other → ignore (modal step)
        ├─ back (closed)       → fall through to Shell leave
        ├─ select              → findProgramAt → detail panel
        └─ arrows              → move cursor → computeVisibleWindow → RAF draw

Whiteboard talking points

  • Always one visible focus target — TV remotes feel broken if nothing is highlighted. Thicker amber stroke + HUD focused: {title} make the demo obvious from 10 feet.
  • Why not DOM focus on canvas? Cells are pixels. Spatial-nav libraries and tabIndex help DOM trees; here the lattice is mathematical.
  • Visible Window still owns draw cost — focus moves recenter the window (Story 2.1 math). You never redraw all 600 programs to move focus.
  • Shared key map (AD-4) — Surfaces map up|down|left|right|select|back; they do not fork raw key strings. Enter is select; Backspace/Escape are back.
  • Nested Back — peel one layer at a time (detail → grid → menu). Capture + stopPropagation only while detail is open so Shell still owns Surface-root leave.

Demo script (desktop proxy)

  1. Shell → focus EPG → Enter.
  2. ↑↓ / ←→ — amber cell moves; HUD shows focused: …; drawn stays ≪ 600.
  3. Enter — detail panel shows title + start/end UTC.
  4. Back — detail closes; still on EPG.
  5. Back again — menu; Surface unmounts. Remount to prove cleanup.
  6. Honesty: Chromium keyboard proxy, not OEM focus engine or TV heap truth.

API shape to remember

  • findProgramAt(channelId, timeMs, programs) — half-open interval match (same as paint).
  • adjacentProgramFocusTime(…, 'left'|'right') — next/prev program start on the channel.
  • Detail panel: DOM sibling under .epg-canvas, data-testid="epg-detail" for future Playwright (7.2).

Honesty bounds

  • Desktop Chromium + keyboard ≠ Tizen/webOS remote or vendor focus frameworks.
  • Program-boundary nav is a teaching choice; some EPGs scrub by fixed time steps instead.
  • Vitest / Playwright for focus helpers land in Epic 7 — keep helpers pure for later.
