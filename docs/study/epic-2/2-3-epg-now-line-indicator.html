EPG · Now-line as separated chrome

The now-line is chrome, not another program cell. Clock ticks move a thin overlay; focus / resize / detail still dirty the Canvas grid. That split is the interview point (AD-5): update current time without rebuilding the logical 600-cell model.

Mental model

  1. Chrome vs cells — Program cells encode schedule data. The now-line encodes “where is demo time on this scale?” Same ms→px formula as cells; different update owner.
  2. Two dirty paths — Grid dirty: focus, Visible Window, resize, detail → programsInVisibleWindow + Canvas paint. Clock dirty: timer tick → update demoNowMs + reposition DOM overlay. No fixture rebuild on ticks.
  3. Demo clock — Fixtures freeze 2026-07-22T00:00:00.000Z for 24h. Raw Date.now() sits outside that day, so the line would never show. Seed with dayStart + (wallMs % scheduleMs), then accelerate (~60s of schedule per 1s wall tick) so learners can see drift. Production EPGs are usually 30–60s / minute-aligned and 1× realtime — say that out loud.
  4. Hide, don’t chase — If demo-now is outside the Visible Window time range, hide the line. No continuous jump-to-now. Mount seeds focus near demo-now once so the line is visible (demo convenience, not a user jump-to-now control).

Whiteboard talking points

  • Distinct affordance — soft-red 2px + “NOW” label. Do not confuse with cyan focus-time cursor or amber focus cell.
  • Same scale — nowX = gutter + ((nowMs − window.timeStart) / MS_PER_HOUR) * pxPerHour. If the formula drifts from cells, the line lies.
  • Why DOM overlay? Interview-strong: ticks never touch Canvas. Android TV Live TV’s ProgramGuide similarly repositions a current-time indicator on a Handler tick without rebuilding program rows.
  • AD-6 — Timer is a Surface side effect. Clear on unmount; include it in hasActiveSideEffects() so Shell cleanup probes stay honest (memory soak later).
  • Teaching vs production tick — Lab ticks every ~1s and advances ~60s of schedule so the line visibly drifts. Production EPGs often tick every 30–60s or minute-align at 1× wall time. Honesty: accelerated demo clock ≠ live airtime.

Demo script (desktop proxy)

  1. Shell → EPG → Enter. Red NOW line should appear (focus seeded near demo-now once).
  2. Wait 1–2 seconds — HUD ticks increments; line visibly drifts (~60×); model stable.
  3. ←→ / ↑↓ — amber focus + cyan cursor move; grid repaints; now-line stays a separate layer.
  4. Enter / Back — detail layering unchanged from 2.2.
  5. Back to menu — status should show side effects cleared; remount restarts the timer.

API shape to remember

  • demoNowMs(dayStartMs, scheduleMs, wallMs?) — remap wall into fixture day.
  • nowXPx(nowMs, window, gutterPx, pxPerHour, msPerHour) — returns null outside the Visible Window (caller hides chrome).
  • Overlay: data-testid="epg-now-line" under .epg-canvas__stage (Playwright later in 7.2).

Honesty bounds

  • Not live schedule sync, OEM EPG, or broadcast “wall clock” truth — mock fixtures only.
  • Accelerated demo clock (~60×) and one-shot focus seed near demo-now are teaching aids — not production jump-to-now or realtime airtime.
  • Jump-to-now control / continuous auto-scroll deferred; Perf Note FPS write-up is Story 2.4.
  • Desktop Chromium + keyboard proxy ≠ Tizen/webOS remote timing or TV heap truth.
  • Vitest for now-line helpers lands in Epic 7 — keep helpers pure.
