EPG · Measured Perf Note

Unlabeled “60 FPS” is a credibility fail. This story makes performance first-class docs: environment + method + numbers under docs/perf-notes/epg.md, linked from the README.

Mental model

  1. AD-8 — Perf Notes live in docs/perf-notes/ with browser / OS / machine class labeled. README links them. Claims without labels do not count.
  2. What to measure on EPG — Primary: drawn ≪ logical (virtualization evidence). Secondary: interaction smoothness / Frame rendering stats while D-pad scrolling. Optional: CPU 4×–6× throttle.
  3. Paint model matters — EPG coalesces Canvas paints on focus / resize / detail. It is not a continuous full-grid RAF loop. Idle compositor FPS ≈ display refresh; do not confuse that with “we paint 120 grids per second.”
  4. Two paths (ties 2.1–2.3) — Focus → Visible Window → paint. Clock tick → now-line chrome only (model stable). Measure both stories honestly. 
logical cells = 600
drawn cells   = tens (HUD)     ←── interview proof of Visible Window

idle rAF      ≈ display Hz     ←── cadence ceiling, not grid work
arrow spam    → schedulePaint  ←── real interaction cost

now-line tick → DOM left only  ←── must not rebuild fixtures

How to measure (Chrome / Chromium)

  1. Run Shell → Menu → EPG.
  2. Read HUD: drawn N ≪ logical 600.
  3. DevTools Command Menu → Show Rendering → enable Frame rendering stats. Hold arrows; watch the overlay.
  4. Optional: Performance panel short recording; optional CPU throttle 4×–6× ([testing-strategy.md](../../testing-strategy.md)).
  5. Write the note with environment labeled — copy the shape in epg.md.
