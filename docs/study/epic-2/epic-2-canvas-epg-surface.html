Canvas EPG Surface

One start→end picture of Lab A: how a large channel × time schedule becomes a demoable Canvas Program Grid — virtualized, focusable, with separated now-line chrome, and a labeled Perf Note so interview claims are measured.

The problem this epic solves

A 50-channel × 24h EPG is ~600 logical program cells. Drawing all of them (or building a DOM node per cell) fails on constrained TV-like budgets. Epic 2 proves the interview answer: compute a Visible Window, draw only that subset, measure it, and keep chrome updates off the grid rebuild path.

Demo / interview script (~5 minutes)

  1. Shell → EPG. Point at HUD: drawn ≪ logical 600.
  2. Hold ↑↓ / ←→ — window moves; focus ring stays visible; only tens of cells draw.
  3. Enter → detail; Back → dismiss; Back again → menu (cleanup / side effects cleared).
  4. Watch NOW line drift; say “chrome tick, not grid rebuild.”
  5. Open docs/perf-notes/epg.md — name browser/OS/machine; refuse unlabeled “60 FPS.”

Whiteboard talking points

  • Why not DOM cells? Node count / layout cost on weak devices.
  • Visible Window — algorithm, not a library slogan; shared math for WebGL Lab next.
  • App-owned focus — do not trust document.activeElement on canvas.
  • Separated dirty paths — grid vs now-line chrome (industry pattern too).
  • Measure → claim — AD-8; desktop proxy ≠ OEM TV.

Honesty bounds

  • Desktop Chromium + keyboard-as-D-pad — not Tizen/webOS certification.
  • Demo clock is accelerated teaching time, not live airtime sync.
  • Vitest / Playwright still Epic 7 — confidence today is typecheck + build + smoke + docs.
  • HiDPI, Date.parse precompute, overlapping-program nav edges — deferred; do not expand into Epic 3 unless blocked.
