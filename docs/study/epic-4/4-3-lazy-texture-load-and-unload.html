Lazy texture load and unload

What shipped: placeholder → upgrade inside focus±2 — far tiles stay cheap color; near tiles load tile-sized SVG “FULL” posters; leave clears the registry + Blits quit.

FR-9 / AD-6 placeholder→upgrade focus ± 2 dispose on leave

What actually shipped

  • Policy: TEXTURE_POLICY = 'placeholder-upgrade', TEXTURE_WINDOW = 2
  • Cheap: hashed color Element (4.2) — no Image src
  • FULL: ~200×260 SVG data-URL Image for indices in focus±2 (badge shows FULL vs cheap)
  • HUD: FULL n (peak m) — n stays ≤ 5 while scrubbing all 12
  • Leave: disposeAllTextures() then app.quit(); [home] dispose log; probe false
  • Docs: Texture lifecycle section in packages/home-blits/README.md

Why TV Home dies without this

Decode every poster at full resolution and never free GPU textures → you burn through a mid-range TV’s usable RAM (often hundreds of MB, not desktop GB). VRAM ≠ JS GC — quitting the app and clearing Image srcs is the discipline.

Interview line: “Same AD-6 honesty as Lab W disposeGpu — Blits quit + an explicit upgrade registry, with a focus window so we never keep all FULL posters forever.”

Behavior is real: scrubbing never holds 12 FULL srcs at once. Tiny SVG stand-ins still prove unload — document “full-res” as the pattern you’re practicing.

WhoResponsibility

Demo script

  1. pnpm dev → Home → note HUD FULL 3 (peak 3) at focus 0.
  2. Hold Right across the rail — FULL stays ≤ 5; far badges say cheap.
  3. Console: [home] textures { loadedFull, peakLoaded, … }.
  4. Escape → [home] dispose { disposedTextures, peakLoaded }; status “Side effects cleared.”
  5. Remount ×3 — no growing peak story; probe stays false after leave.
