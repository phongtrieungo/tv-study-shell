What actually shipped

  • Data: @tvshell/shared homeRails → rail-featured (12 × Featured Tile N) — no local duplicate arrays, no fetch
  • Art: fixtures keep posterUrl: ''; Home maps hashed color placeholders keyed by itemId
  • Focus: Blits input.left / input.right (default keymap = ArrowLeft/Right); clamp at 0…n−1
  • Affordance: amber ring + scale + label color — readable at distance / Safe Zone
  • Scroll: rail x offset keeps focused tile on-stage
  • Back: not handled in Blits — Shell owns Escape/Backspace → host.leave(); remount → focus index 0

Mental model (≤30s)

A TV Home is a focus lattice, not a mouse pointer. One visible focus; Left/Right walk a horizontal rail; the camera (rail offset) follows so focus stays on-screen.

Interview line: “Same AD-4 keys as Canvas EPG and Lab W — Blits scene-graph owns the rail focus; Shell still owns Back leave.”

Demo script

  1. pnpm dev → Menu → Home → Enter.
  2. Point at amber ring on Featured Tile 1; hold Right — focus walks; rail scrolls so focus stays visible.
  3. At the last tile, Right does nothing (clamp). Left returns.
  4. Escape → “Side effects cleared.” Enter again → focus resets to index 0.
  5. Optional: open EPG / WebGL Lab to show other Surfaces intact.
