TV Study Shell · Story 1.4

Surface host mount/unmount contract

This story teaches the Thin DOM Shell’s lifecycle API: Surfaces mount into a host element and unmount must dispose what they created (AD-2 / AD-6). A stub package proves cleanup before real Labs arrive.

Why this story matters

TV apps navigate between heavy Surfaces (Canvas, WebGL, Blits, Solid). If leave only hides DOM and leaves timers, RAF loops, listeners, or GPU textures alive, heap and GPU memory grow across route changes. Interviewers ask who owns cleanup — the answer is a documented contract.

  • FR-2 / AD-2: Shell hosts Surfaces and calls mount / unmount.
  • AD-6: Every unmount disposes listeners, timers, RAF, textures/images the Surface created.
  • Interview value: you can whiteboard ownership without claiming a full Memory Soak yet.

Thin Shell vs Surface ownership

The Shell is Vite + TypeScript DOM. It owns navigation chrome, the host element, and the enter/leave calls. The Surface owns its runtime and resources. Shell does not import Blits/Solid/WebGL into chrome. 

menu → select → host.enter(id) → module.mount(hostEl)
surface Back → host.leave() → module.unmount() → menu

Why not “just swap innerHTML”? Swapping DOM does not cancel setInterval, remove addEventListener, stop RAF, or delete WebGL buffers. Explicit unmount is the teaching surface.

AD-6 cleanup checklist

On leave, a Surface should dispose everything it created:

  • DOM event listeners (removeEventListener)
  • Timers (clearInterval / clearTimeout)
  • RAF loops (cancelAnimationFrame)
  • Textures / images / GPU buffers & programs (later Labs)

Story 1.4 proves the contract with @tvshell/surface-stub: a visibility listener + interval are registered on mount and cleared on unmount. Probe: hasActiveSideEffects() should be false after Back.

Error banner honesty

Architecture convention: mount failures show a Shell-level error banner — no silent blank screens. The host catches throw/reject, clears a half-mounted host element, and surfaces the message.

Stub vs real Surfaces

All four menu destinations currently resolve to the same stub via a registry. Later epics swap entries to epg-canvas, webgl-lab, home-blits, and live-solid without rewriting Shell chrome.

Story 4.1 may adjust Blits mount details (iframe vs in-page). Keep the host API generic: mount into an element (or dedicated route root) — do not hard-code iframe now.

Honesty bounds

  • Stub cleanup ≠ Memory Soak (Story 6.2). Soak validates AD-6 across real Surfaces for ~30 minutes.
  • Desktop Chromium + keyboard is still a proxy, not OEM TV memory truth.
  • Select now means mount (unlike Story 1.3). Real Lab behavior arrives in Epics 2–5.

Whiteboard talking points

  1. Who calls mount/unmount? Thin DOM Shell on enter/leave; Surface implements the pair.
  2. What must unmount dispose? Listeners, timers, RAF, textures/images — whatever that Surface created.
  3. How do you prove it today? Stub registers a listener + timer; after Back, hasActiveSideEffects() is false.
  4. What is still deferred? Real Labs, Blits mount strategy (4.1), cross-surface focus hardening (6.1), Memory Soak (6.2).
